27 ### 1. Introduction
28 Our task was to take text from the AQUINT and TimeBank corpora input and output an xml file for each input file with
29  temporal expressions identified and normalized according to the framework laid out in "A Semantically Compositional
30  Annotation Scheme for Time Normalization"[<sup>1</sup>](#references). We developed a package using python which uses
31  the SUTime[<sup>2</sup>](#references) tagger to identify and mark temporal expressions in documents for processing.
32 
33 
34 ### 2. Background
35 The intent of the this annotation scheme is to capture periods of time that are not well covered by currently existing
36 schemes. The most common scheme is the TIMEX3[<sup>3</sup>](#references) system used by
37 TimeML[<sup>4</sup>](#references) developed out of the older TIMEX scheme which was created by DARPA in
38 1995[<sup>5,6</sup>](#references). Ultimately, the annotations generated in this format can be used by other
39 applications to automatically generate useful information, such as a patient's medical timeline from doctor's notes.
40 
41 
42 ### 3. Method
43 We primarily used SUTime to tag relevant temporal data and then wrote our own methods in python to break those tagged
44 phrases out into entities which were linked together as described by Bethard. Our implementation uses a class called
45 t6Entity which stores all of the information for each entity and is able to print it in the correct format. It also
46 allows t6Entities to be compared by their location in the text and type of entity to ensure we do not generate duplicates.
47 
48 Kuzey et al identified four types of expressions that need to be parsed[<sup>7</sup>](#references):
49 > 1. _Explicit temporal expressions_ denote a precise time point or period...
50 > 2. _Relative temporal expressions_ refer to dates that can be interpreted with respect to a reference date...
51 > 3. _Implicit temporal expressions_ refer to special kinds of named events...
52 > 4. _Free-text temporal expressions_ refer to arbitrary kinds of named events or facts with temporal scopes that are
53 merely given by a text phrase but have unique interpretations given the context and background knowledge...
54 
55 Each type of temporal expression presents unique challenges in handling.

60 ### 5. Conclusion
61 This annotation scheme has the potential to be very useful by providing high quality temporal data to downstream
62 applications. Improvements in correctly identifying free-text, ambiguous temporal expressions will continue to be a
63 challenge. We believe that advances in machine learning will improve correct identification of temporal expressions
64 based on the context in which they are found. Even the tense of a verb carries temporal information which may or may
65 not be relevant thus complicating the task of tagging free-text temporal expressions.
